# Сценарий демо — «Прозрачный поток»

Демонстрация для защиты диплома. Полный цикл занимает ~18 минут.

---

## Запуск (за 5 минут до защиты)

```bash
# Терминал 1 — База данных
docker compose up -d

# Терминал 2 — Бэкенд
cd api
npm run db:reset      # DROP + migrate + seed (создаёт демо-задачу в waiting)
npm run dev           # Express на :3001

# Терминал 3 — Фронтенд (из корня репозитория)
npm run dev           # Vite на :5173
```

Открыть в браузере: **http://localhost:5173**

> **Telegram-онбординг (один раз перед защитой):**
> 1. Войти как admin → открыть любую задачу «ЭкоПродукт» → «Запросить у клиента»
> 2. В синей панели нажать «Сгенерировать Telegram-ссылку» → скопировать deep-link
> 3. Открыть ссылку в Telegram → нажать Start → бот ответит «Чат привязан»
> После этого каскадные уведомления будут приходить в Telegram напрямую.

## Сценарий (~18 минут)

### Акт 1 — Вход и ролевая модель (3 мин)

| | |
|---|---|
| **Что показать** | Разные права для PM и Admin |
| **Логин PM** | `pm@adena.local` / `pm123` |
| **Логин Admin** | `admin@adena.local` / `admin123` |

1. Открыть `http://localhost:5173` → показать экран логина
2. Войти как **PM** → обратить внимание: в колокольчике нет демо-кнопок,
   карточки в `done` нельзя откатить назад
3. Выйти → войти как **Admin** → все возможности доступны

---

### Акт 2 — Канбан-доска и управление задачами (4 мин)

4. **Dashboard-таб:** доска проекта ЭкоПродукт, прогресс-бар спринта сверху
5. **Tasks-таб:** все три проекта (ЭкоПродукт, Кофейня Прима, МедСтарт),
   цветные бейджи, фильтр видимости колонок
6. **Drag-and-drop:** перетащить карточку между колонками (backlog → to-do → in-progress)
7. **TaskModal:** открыть задачу → история изменений, комментарии, ответственные, файлы
8. **Mind Map:** переключить вид → граф зависимостей задач
9. **Новая задача:** хоткей `n` → форма создания прямо на доске

---

### Акт 3 — Клиентский цикл «Запрос контента» (6 мин)

10. Открыть задачу в `in-progress` → нажать **«Запросить у клиента»**
    → статус переходит в `waiting`, синяя панель с magic-ссылкой

11. Скопировать ссылку из панели → открыть в **новой вкладке** браузера
    → загружается **GuestUploadPage** без логина:
    название задачи, дедлайн с обратным отсчётом, зона перетаскивания файлов

12. Загрузить 1–2 файла, написать комментарий → нажать **«Отправить материалы»**
    → статус становится `client-uploaded`

13. Вернуться на PM-доску → карточка переехала в колонку `client-uploaded`

14. Открыть TaskModal → видна жёлтая кнопка **«Принять контент (отправить акт клиенту)»**
    → нажать → статус `done`, в колокольчике появляется запись о верификационном письме

15. В дропдауне колокольчика нажать **«📧 Открыть письмо в Ethereal →»**
    → в браузере открывается HTML-акт о приёмке контента

---

### Акт 4 — Каскадные уведомления (5 мин)

> На доске уже есть задача **«📋 Демо: Тексты для раздела «Услуги»»** в `waiting`
> (создана автоматически при `db:reset`).

16. Нажать иконку 🔔 в шапке → открыть дропдаун уведомлений

17. **«Тик сейчас»** → Level 1: дружелюбное уведомление
    → сообщение приходит в Telegram клиенту (или Ethereal-ссылка в дропдауне)

18. **«+3 дня»** → Level 2: напоминание с акцентом на остановку работ
    → Telegram + Email; в дропдауне появляется «📧 Открыть письмо в Ethereal»

19. **«+5 дней»** → Level 3: официальный юридический тон, сдвиг сроков по договору
    → тональность нарастает; клиент понимает серьёзность

20. **«+8 дней»** → `cascade_exhausted`: внутреннее событие для PM,
    уведомление о том, что автоматика исчерпана — нужно ручное вмешательство
