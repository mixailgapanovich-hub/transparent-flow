## Project Overview

**"Прозрачный поток"** (Transparent Flow) — frontend-only Kanban demo for a digital agency task management system. Built as a diploma thesis prototype for **АденаДиджитал**, a digital agency that manages creative/web projects for clients.

**Scope:** Pure frontend demo. No backend, no API calls, no real auth. All data is in-memory mock state. The goal is a polished, interactive UI that demonstrates the concept: modals, drag-and-drop, mind map, status workflows.

**Business problem it solves:** Agency teams lose track of client tasks across email chains. This dashboard gives PMs a central board; clients get a read-only "magic link" to see their task status and upload materials without needing an account.

## Architecture

### State

All state lives in `src/App.jsx` via `useState`. No Redux, no Zustand, no Context API.

```js
// App.jsx state
isAdmin        // boolean — true by default; unlocks admin status rollbacks
activeTab      // 'dashboard' | 'projects' | 'kb' | 'tasks'
tasks          // Task[] — full task list, initialized from INITIAL_TASKS
activeId       // dnd-kit active drag item id
selectedTaskId // string | null — opens TaskModal when set
isSettingsOpen // boolean — SettingsModal visibility
```

`selectedTask` is a derived value: `tasks.find(t => t.id === selectedTaskId) ?? null`.

### Layout

```
┌──────────┬─────────────────────────────┬──────────────┐
│ Sidebar  │  Header + View              │  RightPanel  │
│  96px    │  (flex-1)                   │  20–64px     │
└──────────┴─────────────────────────────┴──────────────┘
```

Modals (`TaskModal`, `SettingsModal`) render as conditional overlays from `App.jsx` — not portals, just conditionally mounted JSX at the end of the tree.

### Tab/view routing

`activeTab` drives which view is shown in the main area:

| activeTab | Component | Notes |
|-----------|-----------|-------|
| `'dashboard'` | `KanbanBoard` | ✅ Only `proj-eco` tasks; shows sprint progress bar |
| `'tasks'` | `KanbanBoard` | ✅ All projects; project badge per card; column toggle filter; no progress bar |
| `'projects'` | `ProjectsView` | 🚧 Placeholder/skeletal |
| `'kb'` | `KnowledgeBase` | 🚧 Placeholder/skeletal |

Both Kanban views include a built-in Kanban ↔ Mind Map switcher rendered inside `KanbanBoard`.

**Dashboard filter:** `tasks.filter(t => !t.projectId || t.projectId === 'proj-eco')`

**KanbanBoard props that differ between tabs:**

| Prop | dashboard | tasks tab |
|------|-----------|-----------|
| `showProjectBadge` | false (default) | true |
| `showColumnFilter` | false (default) | true |
| `columns` | omitted → COLUMNS | omitted → COLUMNS |

`showColumnFilter=true` hides the sprint `BoardProgress` bar and shows column visibility toggles. Column visibility state (`hiddenColumnIds`) is local to `KanbanBoard`, not lifted to `App.jsx`.

## Status Workflow (FSM)

Defined in `src/utils/taskWorkflow.js`.

```
backlog     → to-do
to-do       → in-progress | backlog
in-progress → waiting | done | to-do
waiting     → in-progress | done
done        → (terminal — no transitions)
```

**Admin override:** When `isAdmin=true`, tasks in `done` can be rolled back to any status.

**Magic link:** Generated automatically (in mockData transform) when `status === 'waiting'`. Format: `https://client.transparent-flow.app/task/{id}`. This represents the guest client link concept — in the demo it's a hardcoded string.

**Key exports:**
- `canTransitionStatus(fromStatus, toStatus, { isAdmin })` → boolean
- `getAllowedStatuses(fromStatus, { isAdmin })` → string[] (includes current status)

## Drag-and-Drop Notes (@dnd-kit)

- **Columns** use `useDroppable` only — they are drop targets, not sortable items.
- **Task cards** use `useSortable` — sortable within and across columns.
- **Collision detection** — `kanbanCollision` (module-level function in `KanbanBoard.jsx`):
  1. `pointerWithin` first — detects the droppable whose bounds contain the mouse pointer; reliably handles empty columns at any cursor position
  2. Falls back to `closestCenter` if pointer is not inside any droppable (keyboard nav, edge cases)
- **Do not use** `closestCorners` — it measures item corner distances and misses empty columns when the cursor is near the top/bottom of an empty container.
- **Column content divs** have `overflow-y-auto` — each column scrolls independently; the board itself scrolls horizontally. Both use the `custom-scrollbar` CSS class defined in `App.jsx`.

## Theming & Conventions

- **Brand blue:** `#3C50B4`
- **Accent gold:** `#FFD700`
- **Style maps** (column colors, tag badges, status button variants, project badges) → `src/theme/taskStyles.js`. Add new status/tag/project styles here, not inline.
  - `TASK_COLUMN_STYLES` — column header, icon, container colors
  - `TASK_TAG_BADGE` — tag badge colors
  - `TASK_STATUS_BADGE` / `TASK_STATUS_LABEL` / `TASK_STATUS_RING` — status display
  - `PROJECT_BADGE_STYLES` — `{ dot, text, label }` per `projectId`
  - `UI_BUTTON_STYLES` — primary / secondary / ghost button variants
- **UI language:** Russian — all labels, tags, status names, date formatting (`ru-RU` locale) are hardcoded in Russian.
- **Hotkeys:** `n` (outside inputs) opens create-task flow; `Esc` closes modals.
- **No TypeScript** — plain JSX throughout. Don't add TS without asking.
- **No test files** — no testing framework configured.
- **Comments in code:** Minimal. Only add if the reason is non-obvious.
